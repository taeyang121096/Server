브라우저 동작 방법

주요 기능 : 사용자가 선택한 자원을 서버에 요청, 브라우저에 표시

자원은 html, pdf, image 등 다양한 형태

자원의 주소는 URI에 의해 정해진다.

브라우저는 html과 css 명세에 따라 html 파일을 해석해서 표시함

명세는 웹 표준화 기구인 W3C에서 정해짐

브라우저가 가진 인터페이스는 보통 비슷비슷한 요소들이 존재

URI 입력하는 주소 표시 줄
이전 버튼, 다음 버튼
북마크(즐겨찾기)
새로 고침 버튼
홈 버튼

사용자 인터페이스 - 브라우저 엔진 ->(자료 저장소) - 렌더링 엔진 (통신, 자바스크립트 해석기, ui백엔드) 

웹킷 동작 구조 : 

파싱 : 문서 파싱은, 브라우저가 코드를 이해하고 

URI : 유니폼 리소스 식별자, 통일적이란 규칙을 따르며 다르것을 구별하는 식별자를 통해 주소를 표현한다.

URI를 사용하면 웹상에 존재하는 모든 리소스를 한결같은 방식으로 보여줌 -> 모든 리소스에 간단하게 접속 가능

URI는 스키마로 시작 되고 호스트명과 패스로 이루어져 있다.

URI는 알파벳, 숫자, 기호로 이루어져있다.
